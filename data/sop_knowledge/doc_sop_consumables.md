Consumables

Captain has not received the consumables OR Cannot order consumables

Scenario 2: Partner name, mobile no, email ID, Hub ID.

Scenario 3: Order ID, Hub ID

Scenario 4: Order ID, Consumables ordered, video proof of unpacking of the consumables.

Before solving for the Scenarios the agent must check if the captain has provided Order ID and Hub ID to simplify the verification process.

If any of the information required per the scenarios is unavailable then reply to the partner following this template.

Agent should follow the instructions when faced with the following scenarios:

Scenario 1 – Captain wants to know how to place an order:

Agent must reply to the partner with the process following this template.

Scenario 2 – Captain is unable to order consumables since they are not whitelisted:

Agent must check if all the required details are present in the ticket. If not then the agent must reply to the partner with the following template asking the partner for the necessary information.

Agent must upload the details onto the whitelisting sheet and reply to the captain stating completion following this template.

Scenario 3 – Captain wants to track their order / Haven’t received their order on time / wants to know the status of their order:

Agent must look up the order ID  mentioned by the partner on the [RVP consumables partner details](https://docs.google.com/spreadsheets/d/1_ghgg3DNq12N2dTma2zkdgPkJbtT0H_aiEegvtB1LSc/edit?gid=862776892#gid=862776892)  by applying a filter  or using Ctrl+F.

Agent must view the corresponding values in the columns - Supplier Code, Dispatch Date, Delivery Date, Order Status, Docket no (which is the tracking id)   and the  courier name.

If the Order Status = Delivered then loop in the Supplier with a valid remark (Direct L3 escalation) .

If order status = dispatched , agent must visit the  tracking site  of the Partner mentioned in the courier name column and enter the value in the docket no   column to track the parcel.
(For example courier name = Amazon, type “Amazon parcel tracking” on the search engine and enter the tracking ID ).

Reply to the partner by mail mentioning the order status in the website  and telling them how to track it themselves following this template.

Scenario 4 – Captain has received lesser consumables than they have ordered:

Agent must look up the order ID  mentioned by the partner on the [RVP consumables partner details](https://docs.google.com/spreadsheets/d/1_ghgg3DNq12N2dTma2zkdgPkJbtT0H_aiEegvtB1LSc/edit?gid=862776892#gid=862776892)  b y applying a filter  or using Ctrl+F.

Agent must view the following columns correspo nding to the  order ID - Order details, qty and order status.

Agent must cross verify the quantity mentioned by the Partner and the quantity mentioned in the [RVP consumables partner details](https://docs.google.com/spreadsheets/d/1_ghgg3DNq12N2dTma2zkdgPkJbtT0H_aiEegvtB1LSc/edit?gid=862776892#gid=862776892)  sheet. ( order Details amount x qty )

If the  quantity is matching and the status = delivered  then reply to the partner using the following template and loop in the Supplier (Direct L3 escalation).

If the quantity is matching  and the status is dispatched,  then reply to the partner using the following template.

If the quantity is not matching  then loop in the Supplier (Direct L3 escalation).

Secondary QC mismatch – Damaged consumables and incorrect formatting.

Secondary QC mismatches can happen for a few reasons: Damaged consumables, code format mismatches, Tampered packets.

Inputs required: order ID, hub ID

Scenario 1: Video evidence of the damaged consumables.

Agents must check if the required information is provided in the ticket and if the information is valid. If not then reply with the following template. Dispose with awaiting user input disposition.

Agent should follow the instructions when faced with the following scenarios:

Scenario 1 – Damaged consumables received:

Agent must reply to the partner stating the inconvenience and looping in the Supplier (Direct L3 escalation).

Reply following this template.

Scenario 2 – Secondary QC failure:

This occurs due to mismatch in the format of the standard codes on the packets causing failure in their scanning or manual input.

This erroneous code will have flown to all the consumables so the evidence required would be a picture of some of the consumables packets with the code present.

Suppliers must be looped into this conversation (Direct L3 escalation).

Reply to the partner using this template.

For the mentioned billing cycles the partner must check the columns rvp_consumable_amount , rvp_consumable_packet_count  and rvp_consumable_rate. (rate x count = amount).

If the count in the column &lt; count mentioned then escalate to the L3.

If the count in the column = count mentioned then reply to the partner following this template.

After the validation from the L3 this issue will go to the Cost-Ops team.

For the mentioned billing cycles the partner must check the columns rvp_consumable_amount , rvp_consumable_packet_count  and rvp_consumable_rate. (rate x count = amount).
