Answer the four questions below in plain English. Click Extract — Gemini converts your answers to structured knowledge, shows you what it understood, and asks specific follow-up questions for anything missing. Click Activate when you're satisfied.

List every label Kapture or captains might use (one per line). E.g. M_V, Payments, payments.
For each issue type describe: what the captain typically writes, what you check to verify, and what reply you send. E.g. "Payment not received: captain says 'salary not credited'. We check fnf_dashboard.payment_process_date — if blank, payment not initiated, reply with timeline. If filled and remark says Hold Negative, escalate to finance."
For each query: name, column list, what each column tells you, and which values map to which action. E.g. "captain_payment_status_and_amount: hub_code (captain's hub), enbolt_id (invoice ref), payment_status (PAID/PENDING/HOLD/PROCESSING), remarks (free-text reason). If status=HOLD and remarks mentions GST → tell captain to clear GST first."
Pre-checks, hard never-do's, mandatory data lookups. Leave blank if none. E.g. "Always check fnf_dashboard before captain_payment_status. Never quote payment dates without confirming via remarks. If enbolt_id missing from ticket, ask captain first."
